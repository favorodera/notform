`NotForm`{lang="ts-type"} is a wrapper component that manages the form lifecycle and provides the form context to nested components like `NotField`{lang="ts-type"} and `NotMessage`{lang="ts-type"}.

## Usage

```vue [form.vue]
<script setup lang="ts">
const { id, submit } = useNotForm({ schema })
</script>

<template>
  <NotForm :id="id" @submit="submit">
    <!-- Form fields go here -->
    <button type="submit">Submit</button>
  </NotForm>
</template>
```

## API

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `id` | `string`{lang="ts-type"} | **Required** | The unique form identifier obtained from `useNotForm`{lang="ts-type"}. |
| `as` | `string`{lang="ts-type"} | `'form'`{lang="ts-type"} | The HTML tag the component should render as. |
| `asChild` | `boolean`{lang="ts-type"} | `false`{lang="ts-type"} | Whether to render the component as its nearest child (renderless). |

### Slots

#### `default`{lang="ts-type"}

The default slot provides access to the entire form context.

**Scope:**

| Property | Type | Description |
| --- | --- | --- |
| `state` | `Ref<Input>`{lang="ts-type"} | Reactive form data. |
| `errors` | `Ref<Issue[]>`{lang="ts-type"} | Reactive validation issues. |
| `isValid` | `ComputedRef<boolean>`{lang="ts-type"} | Form validity status. |
| `isDirty` | `ComputedRef<boolean>`{lang="ts-type"} | Form modification status. |
| `submit` | `Function`{lang="ts-type"} | Form submission handler. |
| `reset` | `Function`{lang="ts-type"} | Form reset handler. |
| `...` | `...` | All other properties returned by [`useNotForm`{lang="ts-type"}](/composables/use-not-form). |
