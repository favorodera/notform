`NotMessage`{lang="ts-type"} is a convenient component that listens to validation errors for a specific field and renders them automatically. It's smart enough to only show the first relevant error message.

## Usage

```vue [form.vue]
<template>
<NotField name="email" v-slot="{ methods, name }">
  <input v-model="state.email" v-bind="methods" :id="name" />

  <!-- NotMessage will automatically show errors for 'email' -->
  <NotMessage name="email" />
</NotField>
</template>
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string`{lang="ts-type"} | **Required** | The path to the field in the form state. |

## Slots

### `default`{lang="ts-type"} 

You can use the default slot to customize how the message is rendered.

**Scope:**

| Property | Type | Description |
| --- | --- | --- |
| `message` | `string \| undefined`{lang="ts-type"}  | The first validation error message for the field. |

**Example:**

```vue [form.vue]
<template>
<NotMessage name="email" v-slot="{ message }">
    <span v-if="message" class="text-red-500 text-sm">
      <Icon name="i-tabler-alert-triangle" class="inline w-4 h-4 mr-1" />
      {{ message }}
    </span>
</NotMessage>
</template>
```

## Internal Logic

- `NotMessage`{lang="ts-type"}  uses `getFieldErrors`{lang="ts-type"}  from the current form context.
- It reactively updates whenever the `errors`{lang="ts-type"}  array for the specified `name`{lang="ts-type"}  changes.
- If no errors exist, it renders nothing.
