`NotField`{lang="ts-type"} is a renderless component that provides field-specific state and validation methods to its children.

## Usage

```vue [form.vue]
<template>
<NotField name="email" v-slot="{ methods, name, errors }">
  <label :for="name">Email</label>
  <input
    v-model="state.email"
    :id="name"
    :name="name"
    v-bind="methods"
  />
  <span v-if="errors.length" class="error">{{ errors[0] }}</span>
</NotField>
</template>
```

## API

### Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string`{lang="ts-type"} | **Required** | The path to the field in the form state (e.g., `'email'`, `'user.profile.name'`). |

### Slots

#### `default`{lang="ts-type"}

The default slot provides access to the field's context.

**Scope:**

| Property | Type | Description |
| --- | --- | --- |
| `name` | `string`{lang="ts-type"} | The field name/path. |
| `errors` | `string[]`{lang="ts-type"} | Active validation messages for this field. |
| `isValid` | `boolean`{lang="ts-type"} | Whether the field is currently valid. |
| `isDirty` | `boolean`{lang="ts-type"} | Whether the field has been modified. |
| `isTouched` | `boolean`{lang="ts-type"} | Whether the field has been focused/blurred. |
| `methods` | `object`{lang="ts-type"} | Event handlers for validation triggers. |
| `validate` | `() => Promise<StandardSchemaV1.Result<ObjectSchema>>`{lang="ts-type"} | Manually trigger validation for this field. |

#### `methods` Scope

| Method | Description |
| --- | --- |
| `onBlur`{lang="ts-type"} | Call when the input loses focus. |
| `onFocus`{lang="ts-type"} | Call when the input gains focus. |
| `onInput`{lang="ts-type"} | Call on every keystroke/input change. |
| `onChange`{lang="ts-type"} | Call when the input value is committed. |

::tip
Always `v-bind="methods"` to your input element to ensure validation triggers work as configured in `useNotForm`.
::
