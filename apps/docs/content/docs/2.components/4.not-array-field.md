`<NotArrayField>` manages an array within your form values. It exposes items with stable keys for `v-for`, array state, and mutation methods through its default slot. Item keys survive reorders, inserts, and removals — elements are not re-mounted unnecessarily.

## Props

::field-group
  ::field{name="path" type="string" required}
  Dot-separated path to the array within `form.values`.
  ::
  ::field{name="itemSchema" type="StandardSchemaV1"}
  Schema for a single array item. Used only for TypeScript inference — it enables typed `append`, `prepend`, `insert`, and `update` in the slot. Not used for runtime validation.
  ::
  ::field{name="validateOn" type="Partial<Record<'onMount' | 'onChange', boolean>>"}
  Trigger overrides for this array. Only `onMount` and `onChange` are supported — `onBlur`, `onFocus`, and `onInput` are not meaningful at the array level.
  ::
  ::field{name="form" type="NotFormInstance<any>"}
  Explicit instance override. Required when used outside of `<NotForm>`.
  ::
::

## Slot props

### State

::field-group
  ::field{name="path" type="string"}
  The dot-separated path passed to this array field.
  ::
  ::field{name="items" type="NotArrayFieldItem[]"}
  The current items. Each entry has a stable `key`, the current `index`, and the full `path` to pass to a nested `<NotField>`.
  ::
  ::field{name="length" type="number"}
  The current array length.
  ::
  ::field{name="errors" type="StandardSchemaV1.Issue[]"}
  Validation issues targeting this array path specifically.
  ::
  ::field{name="isValid" type="boolean"}
  `true` when there are no errors for this array.
  ::
  ::field{name="isTouched" type="boolean"}
  `true` if the array path itself or any item path has been touched.
  ::
  ::field{name="isDirty" type="boolean"}
  `true` if the array path itself or any item path is dirty.
  ::
  ::field{name="isValidating" type="boolean"}
  `true` while validation is running for this array.
  ::
  ::field{name="validate" type="() => Promise<StandardSchemaV1.Result>"}
  Manually triggers validation for the array path.
  ::
::

### Mutation methods

::field-group
  ::field{name="append(value)" type="(value: TItem) => void"}
  Adds an item to the end of the array.
  ::
  ::field{name="prepend(value)" type="(value: TItem) => void"}
  Adds an item to the beginning of the array.
  ::
  ::field{name="remove(index)" type="(index: number) => void"}
  Removes the item at the given index.
  ::
  ::field{name="insert(index, value)" type="(index: number, value: TItem) => void"}
  Inserts an item at the given index. Subsequent items shift forward.
  ::
  ::field{name="update(index, value)" type="(index: number, value: TItem) => void"}
  Replaces the value at the given index. The item's stable key is preserved.
  ::
  ::field{name="swap(indexA, indexB)" type="(indexA: number, indexB: number) => void"}
  Swaps two items. Their stable keys move with them.
  ::
  ::field{name="move(from, to)" type="(from: number, to: number) => void"}
  Moves an item from one index to another. Items between the positions shift to fill the gap.
  ::
::

## The `NotArrayFieldItem` type

```ts
type NotArrayFieldItem = {
  key: string   // Stable — does not change on reorder
  index: number // Current position in the array
  path: string  // Full dot-separated path — pass directly to <NotField>
}
```

Always use `item.key` on `v-for` and `item.path` as the `path` prop on nested `<NotField>` components.

## Usage

### String array

::code-preview
  ::ArrayFieldDemo
  ::

  #code
  ```vue
  <script setup lang="ts">
  const form = useNotForm({
    schema: z.object({ tags: z.array(z.string().min(1)) }),
    initialValues: { tags: ['vue'] },
  })
  </script>

  <template>
    <NotForm :form="form" @submit="form.submit">
      <NotArrayField path="tags" v-slot="{ items, append, remove }">
        <div v-for="item in items" :key="item.key" class="flex gap-2">
          <NotField :path="item.path" v-slot="{ events }">
            <input v-model="form.values.tags[item.index]" v-bind="events" />
            <NotMessage :path="item.path" />
          </NotField>
          <button type="button" @click="remove(item.index)">Remove</button>
        </div>
        <button type="button" @click="append('')">Add tag</button>
      </NotArrayField>
      <button type="submit">Save</button>
    </NotForm>
  </template>
  ```
::

### Typed items with `itemSchema`

Pass `itemSchema` to make `append`, `prepend`, `insert`, and `update` typed. Without it they accept `any`.

::code-collapse
```vue
<script setup lang="ts">
const itemSchema = z.object({
  name: z.string().min(1),
  role: z.enum(['admin', 'user']),
})

const form = useNotForm({
  schema: z.object({ members: z.array(itemSchema) }),
  initialValues: { members: [] },
})
</script>

<template>
  <NotForm :form="form" @submit="form.submit">
    <NotArrayField path="members" :item-schema="itemSchema" v-slot="{ items, append, remove }">
      <div v-for="item in items" :key="item.key">
        <NotField :path="`${item.path}.name`" v-slot="{ events }">
          <input v-model="form.values.members[item.index].name" v-bind="events" />
          <NotMessage :path="`${item.path}.name`" />
        </NotField>

        <NotField :path="`${item.path}.role`" v-slot="{ events }">
          <select v-model="form.values.members[item.index].role" v-bind="events">
            <option value="admin">Admin</option>
            <option value="user">User</option>
          </select>
        </NotField>

        <button type="button" @click="remove(item.index)">Remove</button>
      </div>

      <!-- TypeScript knows the shape: { name: string; role: 'admin' | 'user' } -->
      <button type="button" @click="append({ name: '', role: 'user' })">Add member</button>
    </NotArrayField>
  </NotForm>
</template>
```
::

### Reordering with move

```vue
<NotArrayField path="steps" v-slot="{ items, move, remove }">
  <div v-for="item in items" :key="item.key" class="flex items-center gap-2">
    <NotField :path="item.path" v-slot="{ events }">
      <input v-model="form.values.steps[item.index]" v-bind="events" />
    </NotField>
    <button @click="move(item.index, item.index - 1)" :disabled="item.index === 0">Up</button>
    <button @click="move(item.index, item.index + 1)" :disabled="item.index === items.length - 1">Down</button>
    <button @click="remove(item.index)">Remove</button>
  </div>
</NotArrayField>
```

### Array-level errors

Issues targeting the array path itself (e.g., minimum length rules) are in `errors`, separate from individual item errors.

```vue
<NotArrayField path="tags" v-slot="{ items, errors, append, remove }">
  <!-- items -->
  <p v-if="errors.length" class="text-sm text-red-500">{{ errors[0].message }}</p>
  <button @click="append('')">Add tag</button>
</NotArrayField>
```
