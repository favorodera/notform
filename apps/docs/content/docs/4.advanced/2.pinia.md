A form instance is a plain reactive object. You can store it in a Pinia store, return it from a composable, or pass it around like any other reactive value.

## Scoped composable

The simplest pattern. One composable creates and owns an instance for a specific feature. Any component that calls it gets the same state when the composable is called within the same scope.

```ts [composables/useContactForm.ts]
import { z } from 'zod'

const schema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  message: z.string().min(10),
})

export function useContactForm() {
  return useNotForm({
    schema,
    initialValues: { name: '', email: '', message: '' },
    onSubmit: async (values) => {
      await $fetch('/api/contact', { method: 'POST', body: values })
    },
  })
}
```

```vue [ContactForm.vue]
<script setup lang="ts">
const form = useContactForm()
</script>
```

::warning
Each call to `useContactForm()` creates a new instance. To share state between two components, call it once in a parent and pass the instance down as a prop — or use a Pinia store.
::

## Pinia store

Store the instance in a Pinia store for global access across a complex feature.

```ts [stores/signup.ts]
import { defineStore } from 'pinia'
import { z } from 'zod'

const schema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  password: z.string().min(8),
})

export const useSignupStore = defineStore('signup', () => {
  const form = useNotForm({
    schema,
    initialValues: { name: '', email: '', password: '' },
    onSubmit: async (values) => {
      await $fetch('/api/signup', { method: 'POST', body: values })
      await navigateTo('/dashboard')
    },
  })

  return { form }
})
```

```vue [SignupStep1.vue]
<script setup lang="ts">
const { form } = useSignupStore()
</script>

<template>
  <NotField :form="form" path="name" v-slot="{ events }">
    <input v-model="form.values.name" v-bind="events" />
    <NotMessage :form="form" path="name" />
  </NotField>
</template>
```

```vue [SignupStep2.vue]
<script setup lang="ts">
const { form } = useSignupStore()
</script>

<template>
  <NotField :form="form" path="email" v-slot="{ events }">
    <input v-model="form.values.email" v-bind="events" />
    <NotMessage :form="form" path="email" />
  </NotField>
</template>
```

::note
`useNotForm` uses `reactive()` internally. Pinia does not interfere with the arrays and Sets inside the instance. Values, errors, `touchedFields`, and `dirtyFields` remain fully reactive. The instance itself is wrapped with `markRaw` to prevent Pinia from making it deeply reactive unnecessarily.
::

## Multi-step form

A single instance spanning multiple steps. Each step validates only its own fields before advancing.

::code-collapse
```ts [stores/onboarding.ts]
import { defineStore } from 'pinia'
import { z } from 'zod'

const schema = z.object({
  firstName: z.string().min(1),
  lastName: z.string().min(1),
  company: z.string().min(1),
  role: z.string().min(1),
  plan: z.enum(['free', 'pro', 'enterprise']),
})

export const useOnboardingStore = defineStore('onboarding', () => {
  const form = useNotForm({
    schema,
    initialValues: {
      firstName: '', lastName: '',
      company: '', role: '',
      plan: 'free',
    },
    onSubmit: async (values) => {
      await $fetch('/api/onboarding', { method: 'POST', body: values })
    },
  })

  async function validateStep(fields: string[]) {
    const results = await Promise.all(
      fields.map(f => form.validateField(f as any))
    )
    return results.every(r => !r.issues)
  }

  return { form, validateStep }
})
```

```vue [Step1.vue]
<script setup lang="ts">
const { form, validateStep } = useOnboardingStore()

async function next() {
  const valid = await validateStep(['firstName', 'lastName'])
  if (valid) navigateTo('/onboarding/step-2')
}
</script>

<template>
  <NotField :form="form" path="firstName" v-slot="{ events }">
    <input v-model="form.values.firstName" v-bind="events" />
    <NotMessage :form="form" path="firstName" />
  </NotField>
  <NotField :form="form" path="lastName" v-slot="{ events }">
    <input v-model="form.values.lastName" v-bind="events" />
    <NotMessage :form="form" path="lastName" />
  </NotField>
  <button @click="next">Next</button>
</template>
```
::

## Reading state outside components

The instance is plain reactive — read it in watchers, computed values, or other stores.

```ts
const store = useSignupStore()

// Watch for any dirty field
watch(() => store.form.isDirty.value, (dirty) => {
  if (dirty) enableDraftSave()
})

// Check validity without running validation
if (store.form.isValid.value) {
  proceedToPayment()
}
```
