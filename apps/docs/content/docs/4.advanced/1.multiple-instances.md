Every call to `useNotForm` creates an isolated instance. There is no global state. Submitting, validating, or resetting one form has no effect on any other.

## Two forms on one page

Each instance tracks its own values, errors, touched fields, and submission state independently.

::code-collapse
```vue [settings.vue]
<script setup lang="ts">
const profileForm = useNotForm({
  schema: z.object({
    name: z.string().min(2),
    bio: z.string().max(200),
  }),
  initialValues: { name: '', bio: '' },
  onSubmit: (values) => saveProfile(values),
})

const passwordForm = useNotForm({
  schema: z.object({
    current: z.string().min(8),
    next: z.string().min(8),
    confirm: z.string(),
  }).refine(v => v.next === v.confirm, {
    message: 'Passwords do not match',
    path: ['confirm'],
  }),
  initialValues: { current: '', next: '', confirm: '' },
  onSubmit: (values) => changePassword(values),
})
</script>

<template>
  <NotForm :form="profileForm" @submit="profileForm.submit">
    <NotField path="name" v-slot="{ events }">
      <input v-model="profileForm.values.name" v-bind="events" />
      <NotMessage path="name" />
    </NotField>
    <button type="submit">Save profile</button>
  </NotForm>

  <NotForm :form="passwordForm" @submit="passwordForm.submit">
    <NotField path="current" v-slot="{ events }">
      <input v-model="passwordForm.values.current" type="password" v-bind="events" />
      <NotMessage path="current" />
    </NotField>
    <button type="submit">Change password</button>
  </NotForm>
</template>
```
::

## Passing an instance as a prop

Pass the instance as a prop when a child component owns part of the form UI. The child calls the same API without needing its own instance.

```ts [types]
// Child component prop type
defineProps<{
  form: NotFormInstance<typeof schema>
}>()
```

```vue [NameField.vue]
<script setup lang="ts">
const props = defineProps<{ form: NotFormInstance<any> }>()
</script>

<template>
  <NotField :form="props.form" path="name" v-slot="{ events }">
    <input v-model="props.form.values.name" v-bind="events" />
    <NotMessage :form="props.form" path="name" />
  </NotField>
</template>
```

```vue [parent.vue]
<script setup lang="ts">
const form = useNotForm({ schema, initialValues: { name: '' } })
</script>

<template>
  <NotForm :form="form" @submit="form.submit">
    <NameField :form="form" />
    <button type="submit">Save</button>
  </NotForm>
</template>
```

## `:form` prop priority

When a field component is inside a `<NotForm>` ancestor but also receives `:form` directly, **the `:form` prop always wins**. This lets you place a field from one form inside the markup of another without conflict.

```vue
<NotForm :form="primaryForm" @submit="primaryForm.submit">
  <!-- This field belongs to secondaryForm -->
  <NotField :form="secondaryForm" path="coupon" v-slot="{ events }">
    <input v-model="secondaryForm.values.coupon" v-bind="events" />
  </NotField>
</NotForm>
```

## Dynamic list of forms

Create instances in a reactive array when you need one form per row.

::code-collapse
```vue
<script setup lang="ts">
const rowSchema = z.object({
  name: z.string().min(1),
  qty: z.number().min(0),
})

const rows = reactive(
  [
    { name: 'Widget A', qty: 1 },
    { name: 'Widget B', qty: 3 },
  ].map(initialValues =>
    useNotForm({ schema: rowSchema, initialValues, onSubmit: saveRow }),
  ),
)

function addRow() {
  rows.push(
    useNotForm({ schema: rowSchema, initialValues: { name: '', qty: 0 }, onSubmit: saveRow }),
  )
}
</script>

<template>
  <div v-for="(form, i) in rows" :key="i">
    <NotForm :form="form" @submit="form.submit">
      <NotField path="name" v-slot="{ events }">
        <input v-model="form.values.name" v-bind="events" />
      </NotField>
      <NotField path="qty" v-slot="{ events }">
        <input type="number" v-model.number="form.values.qty" v-bind="events" />
      </NotField>
      <button type="submit" :disabled="form.isSubmitting.value">Save</button>
    </NotForm>
  </div>
  <button @click="addRow">Add row</button>
</template>
```
